# Sample Research Outputs

This page contains example outputs generated by the local AI Research Agent demo.
They are included so reviewers can quickly see how the agent plans searches, grounds
answers in sources, and stores research history.

> These are snapshots from local runs. Results can vary because the agent queries live web
> sources and uses free OpenRouter models.

![Research Agent UI](assets/research-agent-ui.png)

## What To Look For

- The agent generates its own search queries before retrieval.
- Non-academic questions avoid noisy arXiv results and use web sources instead.
- Outputs include structured key findings, summaries, and cited source objects.
- Each run is persisted and can be loaded again from the session history.

## Example 1: RAG

**Question**

```text
What is RAG in AI?
```

**Generated search queries**

- `What is RAG in AI?`

**Key findings**

1. RAG, or Retrieval-Augmented Generation, connects an LLM to an external knowledge source and retrieves relevant documents at inference time.
2. It helps reduce hallucinations by grounding answers in fresh, domain-specific information.
3. It can work with structured enterprise data and unstructured text, often using vector search for retrieval.

**Summary snapshot**

Retrieval-Augmented Generation is a framework that enhances large language models by fetching trusted external information before producing a response. The agent identified RAG as a way to ground model outputs in current facts, reduce hallucinations, and avoid expensive model retraining for domain-specific use cases.

**Sources used**

- [What is Retrieval-Augmented Generation (RAG)? A Practical Guide](https://www.k2view.com/what-is-retrieval-augmented-generation)
- [What is retrieval-augmented generation (RAG)? - IBM Research](https://research.ibm.com/blog/retrieval-augmented-generation-RAG?utm_source=ts2.tech)
- [What is Retrieval-Augmented Generation? - AWS](https://aws.amazon.com/what-is/retrieval-augmented-generation/)
- [What is Retrieval-Augmented Generation (RAG)? | Google Cloud](https://cloud.google.com/use-cases/retrieval-augmented-generation)

## Example 2: LangGraph

**Question**

```text
What is LangGraph and when should you use it?
```

**Generated search queries**

- `What is LangGraph`
- `LangGraph features and capabilities`
- `When to use LangGraph use cases`

**Key findings**

1. LangGraph enables stateful, multi-agent workflows through graph-based control flow.
2. It integrates with the LangChain ecosystem while giving developers more explicit control over state and routing.
3. It is useful for cyclical workflows, agent loops, human-in-the-loop steps, and long-running agent processes.

**Summary snapshot**

LangGraph is a framework for building agentic systems where steps are represented as graph nodes and state moves through the workflow. It is most useful when a project needs more than a single prompt call: branching, retries, tool use, multi-step planning, persistence, or multi-agent coordination.

## Example 3: Vector Databases For RAG

**Question**

```text
Compare vector databases for RAG: Pinecone, Weaviate, and Qdrant.
```

**Generated search queries**

- `How do Pinecone, Weaviate, and Qdrant handle similarity search operations?`
- `What are the key differences in scalability and performance between these vector databases?`
- `Which database offers better integration with RAG workflows for large-scale applications?`

**Key findings**

1. Weaviate is open-source and supports hybrid search, metadata-aware retrieval, sharding, and HNSW indexing.
2. Pinecone is a managed cloud vector database focused on production similarity search.
3. Qdrant is open-source, supports cloud and self-hosted deployment, and is commonly used for high-dimensional vector search.

**Summary snapshot**

The agent compared the three tools as vector stores for retrieval-heavy applications. It highlighted Weaviate's hybrid search and scalability features, Pinecone's managed cloud approach, and Qdrant's open-source deployment flexibility. The retrieved evidence positioned all three as suitable for semantic search, recommendation systems, and RAG pipelines.

**Sources used**

- [What Is Weaviate? A Semantic Search Database](https://www.oracle.com/database/vector-database/weaviate/)
- [What is Weaviate? Meaning, Architecture, Examples, Use Cases, and How to Measure It](https://aiopsschool.com/blog/weaviate/)
- [What Is Qdrant? A Vector Search Engine](https://www.oracle.com/database/vector-database/qdrant/)
